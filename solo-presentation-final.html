HSA‑001Why This ExistsSheet 03/16

Testing on a distributed ledger is painful

The old options

  • Public testnet: shared, rate-limited, can't reset, needs internet
  • Hand-built local network: many moving parts, hours of fragile setup

What developers actually want

  • A private, disposable network they fully control
  • Reset to a clean state, fund accounts, run offline, throw away
  • Works the same on a laptop and in CI
Solo makes it one command. hiero‑solo‑action makes it one CI step.
HSA‑001BackgroundSheet 04/16

Hedera vs. Hiero

Hedera

A public distributed-ledger network. Uses Hashgraph consensus (gossip-about-gossip + virtual voting), not a traditional blockchain. Native coin: HBAR.

Hiero

The open-source codebase that powers Hedera, contributed to the Linux Foundation for community governance.

Rule of thumb: Hiero = the software  ·  Hedera = the public network that runs it.

HSA‑001Core ValueSheet 05/16

Why run a network locally?

Control

Reset to genesis, fund accounts with fake HBAR, full admin.

Isolation

No shared testnet, no rate limits, works offline.

Disposable

Spin up, test, tear down, repeat on every change.

If Hedera's public network is grid electricity, Solo is a generator in your garage, and the action is a robot that switches it on automatically.

HSA‑001The Tool ItselfSheet 09/16

What Solo is

"An opinionated CLI tool to deploy and manage standalone test networks."

What it does

Orchestrates consensus nodes, mirror nodes, explorers, and relays via Helm charts on Kind clusters.

Installnpm install -g @hiero-ledger/solo
RequiresNode.js ≥ 22, Docker

Self-managing toolchain

Solo auto-installs kubectl, Helm, and Kind at deploy time, so you only need Docker + Node.

HSA‑002hiero-solo-actionSheet 13/16

The same network, inside CI

A composite GitHub Action you can drop into any workflow:

  • No manual Java / Node / Kind / Helm / Solo setup
  • One step → a live, throwaway network
  • Hands your tests funded account credentials

Default: consensus node only. Enable mirror node and relay via inputs.

- name: Setup Hiero Solo
  uses: hiero-ledger/hiero-solo-action@v0.21.0
  id: solo

- name: Run tests
  env:
    ACCOUNT_ID: ${{ steps.solo.outputs.accountId }}
  run: npm test
